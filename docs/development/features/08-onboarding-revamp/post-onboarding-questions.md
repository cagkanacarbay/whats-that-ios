# Post-Onboarding Questions (Out of Scope for Pre-Onboarding)

These questions need to be addressed when we refine the post-onboarding flow. They are **not** part of the current pre-onboarding work.

## Questions to Resolve

### 1. Voice/Speaker Selection
- When should users choose their preferred narrator voice?
- Should this be part of post-onboarding, or deferred to settings?
- What's the default voice if they never choose?

### 2. Location Permission
- When should we request location permission?
- Options:
  - During post-onboarding (before first discovery)
  - When user taps "Take Photo" for first time
  - When user views their first discovery on map
- Trade-off: Earlier request = more friction, but ensures location data for discoveries

### 3. IPoP (In-Person or Photo) Preference
- Is this still needed?
- If so, when should we ask?
- Can we infer from user behavior instead?

### 4. First Discovery Flow
- Current: Simple "Take a Photo" / "Use Gallery" buttons
- Should we guide them more explicitly?
- Should we show tips/hints for best photo results?

### 5. Empty State
- What does the discoveries page look like with 0 discoveries?
- Should we show a prominent CTA to create first discovery?
- Should we show sample discoveries as inspiration?
