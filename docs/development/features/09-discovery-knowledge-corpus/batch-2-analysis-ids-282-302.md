## INSIGHTS FOR SYSTEM PROMPT IMPROVEMENT

1. **"DEVELOP EVERY MENTION" Rule:** If you name a person, artist, or historical figure, dedicate at least 2-3 sentences to their specific actions, decisions, or characteristics. Current batch name-drops figures (Tiepolo, Cranach, Platzer, Palloni) then abandons them.

2. **Avoid Topic-Per-Section Pattern:** The H2 "header → 6 sentences → next header" structure encourages surface coverage. Instead:
   - Choose ONE primary thread or 2-3 deeply connected aspects
   - Develop each across multiple sections with callbacks
   - Build to a conclusion/crisis/meaning

3. **Mechanism > Assertion:** ID 302 and ID 301 succeed because they explain HOW things work (water slows → minerals settle; thermal water erodes stone slowly). ID 283 (Tiepolo's light) and ID 290 (Hummel's geometry) fail because they assert mastery without showing it.

4. **Environmental/Human Stakes:** ID 302's Section 4 (water diversion crisis) gives the entire piece consequence. Without it, it's just beautiful geology. Consider: what happens if this is lost? Why does it matter NOW?

5. **Superlatives Need Evidence:** Phrases like "master," "most famous," "legendary," "most sought-after" appear in ~60% of batch but are never supported. Either:
   - Show the evidence (what makes them a master?)
   - Or remove the claim and let the work speak

6. **Mode B (Multi-Aspect) Requires Integration:** When covering 3-4 aspects, they should comment on each other or build toward an insight. Current batch treats each section as standalone.
